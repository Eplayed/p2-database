## 优化: Puppeteer 在 GitHub Actions 稳定性

### 问题
- CallbackRegistry ProtocolError
- 内存泄漏导致超时

### 解决方案

#### translate_crawler.js 优化
1. 添加 `isCI` 环境检测
2. 添加 `BROWSER_RESTART_INTERVAL` (CI: 3, 非CI: 10)
3. 添加 `createBrowser()` 函数，含重试逻辑（最多3次）
4. 优化 Chrome args：
   - `--disable-dev-shm-usage` (解决 CI 内存问题)
   - `--disable-memory-info`
   - `--disable-oom-killer`
   - `--disable-background-timer-throttling`
   - `--disable-renderer-backgrounding`
5. 添加 `createPage()` 函数，优化请求拦截
6. 降低分辨率到 1280x720 节省内存
7. 添加 `safeClosePage()` 安全关闭函数
8. 每处理 N 个职业后定期重启浏览器

#### GitHub Actions 优化
1. 添加 job 级别 `timeout-minutes: 25`
2. 添加 step 级别 `timeout-minutes: 20`
3. 增加文件描述符限制到 8192
4. 添加 `vm.swappiness=10` 减少 swap
5. 添加更多系统依赖：
   - `libpangocairo-1.0-0`
   - `libgtk-3-0`
   - `fonts-liberation`
   - `wget`, `gnupg`
6. 显示系统状态（内存、磁盘）
