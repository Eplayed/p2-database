## 修复天赋树黑图和Keystones图标问题

### 问题分析
1. **天赋树黑图**：`page.screenshot` 的 `clip` 参数需要**绝对坐标**，而 `getBoundingClientRect()` 返回的是相对坐标（相对于视口），需要加上 `scrollX/scrollY`
2. **Keystones图标没有**：poe.ninja API 可能不总是返回 keystones 数据，需要添加 DOM 兜底提取逻辑

### 修复方案

#### translate_crawler.js
```javascript
// 修复前（相对坐标，黑图）
x: Math.round(rect.x),
y: Math.round(rect.y),

// 修复后（绝对坐标）
x: Math.round(rect.x + window.scrollX),
y: Math.round(rect.y + window.scrollY),
```

#### keystones DOM 兜底逻辑
当 API 返回的 keystones 为空时，从天赋树区域的 `<img>` 标签中提取 `/keystone/` 路径的图标

### 修改的文件
- `auto_browser/translate_crawler.js` - 截图坐标 + keystones DOM提取
- `auto_browser/auto_full_crawler.js` - 同样的修复
