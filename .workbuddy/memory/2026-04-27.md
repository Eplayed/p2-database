## 修复天赋树黑图和Keystones图标问题

### 问题分析
1. **天赋树黑图**：`page.screenshot` 的 `clip` 参数需要**绝对坐标**，而 `getBoundingClientRect()` 返回的是相对坐标（相对于视口），需要加上 `scrollX/scrollY`
2. **Keystones图标没有**：poe.ninja API 可能不总是返回 keystones 数据，需要添加 DOM 兜底提取逻辑

### 修复方案

#### translate_crawler.js
```javascript
// 修复前（相对坐标，黑图）
x: Math.round(rect.x),
y: Math.round(rect.y),

// 修复后（绝对坐标）
x: Math.round(rect.x + window.scrollX),
y: Math.round(rect.y + window.scrollY),
```

#### keystones DOM 兜底逻辑
当 API 返回的 keystones 为空时，从天赋树区域的 `<img>` 标签中提取 `/keystone/` 路径的图标

### 修改的文件
- `auto_browser/translate_crawler.js` - 截图坐标 + keystones DOM提取
- `auto_browser/auto_full_crawler.js` - 同样的修复

## 小程序BD推荐缓存策略（3天）

### 修改内容
为 BD 推荐数据（community.json）实现 3 天更新一次的缓存策略。

### 修改的文件

#### 1. `src/utils/index.js` - StorageUtils 扩展
新增带过期时间的缓存方法：
- `setWithExpire(key, value, expireDays)` - 设置带过期时间的缓存
- `getWithExpire(key, defaultValue)` - 获取缓存（过期自动返回默认值）
- `isExpired(key)` - 检查缓存是否过期
- `getExpireRemaining(key)` - 获取剩余有效时间

#### 2. `src/pages/index/index.vue` - 首页热门BD
- 新增 `BD_CACHE_KEY = 'poe2_community_cache'` 缓存键
- 修改 `fetchHotBdList()` 函数：
  - 优先从缓存读取（3天有效期）
  - 缓存不存在或已过期时从远程获取
  - 获取后自动存入缓存
  - 加载失败时使用过期缓存作为兜底

#### 3. `src/pages-sub/starter/index.vue` - 开荒推荐页面
- 新增缓存键 `STARTER_CACHE_KEY`、`COMMUNITY_CACHE_KEY`
- 修改 `loadCurrentData()` 函数：
  - 同样使用 3 天过期时间
  - Tab 切换时检查缓存
  - 加载失败时使用过期缓存兜底

### 缓存键
- 首页热门BD：`poe2_community_cache`
- 开荒流派：`poe2_starters_cache`
- BD推荐：`poe2_community_cache`
