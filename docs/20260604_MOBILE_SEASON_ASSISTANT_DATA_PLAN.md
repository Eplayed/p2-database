# 移动端赛季助手数据支持 P0-P2 计划

更新时间：2026-06-04

## 目标

小程序后续重点从“Web 工具替代品”转向“随身赛季助手”。p2-database 需要为首页摘要、移动阅读、收藏和轻量判断提供小而准的数据，不追求把所有原始数据直接下发到小程序。

## P0：支撑首页随身信息流

### 经济摘要

- [ ] 继续生成 `miniprogram_data/economy_digest.json`。
- [ ] 首页只消费摘要字段，不拉全量经济原始表。
- [ ] 为经济物品补充移动端用途标签：
  - 核心汇率。
  - 开荒可卖。
  - 终局门票。
  - 做装材料。
  - 涨幅异常。
  - 高成交量。
- [ ] 为经济物品补充规则建议：
  - 留。
  - 卖。
  - 关注。
- [ ] 英文名称继续补中文翻译，优先补首页和经济页会展示的 50-100 个物品。
- [ ] 图标继续保存到 `miniprogram_data/economy-icons/*`，raw 数据不上传 OSS。

### 热门 BD 摘要

- [ ] 继续维护正式开荒推荐 `starters.json`。
- [ ] 继续维护热门候选 `starter_candidates.json`。
- [ ] 为首页热门 BD 提供轻量字段：
  - 标题。
  - 职业/升华。
  - 主技能。
  - 适合场景。
  - 新手友好。
  - 造价。
  - 操作难度。
  - 来源。
  - 更新时间。
- [ ] Agent 产出仍只作为候选，人工审核后再进入正式推荐。

### 资讯摘要

- [ ] 新闻抓取继续输出首页需要的最新 3-5 条。
- [ ] 标记重要程度：
  - 版本更新。
  - 开服信息。
  - 机制说明。
  - 社区热帖。

## P1：支撑收藏和移动阅读

- [ ] 为 BD、经济物品、Boss、机制资料提供稳定 `id`。
- [ ] 资料条目保持短文本字段，适合手机阅读：
  - 是什么。
  - 为什么重要。
  - 怎么获得。
  - 怎么用。
  - 注意事项。
- [ ] BD 详情补阶段结构：
  - `level_1_20`
  - `level_20_40`
  - `level_40_70`
  - `early_endgame`
- [ ] 对热门 BD 候选增加质量字段：
  - 是否开荒。
  - 是否完整。
  - 是否有技能阶段。
  - 是否有天赋说明。
  - 是否有转职/转型节点。

## P2：支撑工具导航和剧情增强

### 工具导航

- [ ] 新增可维护的工具导航数据源，建议放在 `base-data/tools/`。
- [ ] 输出小程序 JSON：
  - 分类。
  - 名称。
  - 说明。
  - 链接。
  - 是否推荐。
  - 使用提示。
- [ ] 工具导航主要是复制链接，不做复杂 WebView 依赖。

### 剧情地图

- [ ] 修复剧情路线分叉模型，不再只保留一个 `nextPointId`。
- [ ] 点位增加结构化奖励类型：
  - 主线。
  - 天赋点。
  - 抗性。
  - 精神。
  - 最大生命。
  - Boss。
  - 任务物品。
- [ ] 增加章节奖励汇总：
  - 天赋点总数。
  - 抗性奖励。
  - 精神奖励。
  - 生命奖励。
  - 关键任务物品。
- [ ] 增加国服名/国际服名字段。
- [ ] 坐标校验继续放到 Dashboard 或脚本中，避免错误点位直接上线。

## 更新方式

- GitHub Actions 定时继续负责经济数据。
- Dashboard 保留“一键更新 0.5 经济榜”。
- 首页摘要需要优先低下行：
  - 小 JSON。
  - 图片缓存。
  - 详情页按需拉取。
- 任何候选 BD 自动化都不得直接覆盖正式推荐。

## 验收标准

- 小程序首页可用一个经济摘要文件展示今日经济判断。
- 首页热门 BD 不依赖全量帖子正文。
- 资料字段适合手机短阅读。
- OSS 上传不包含 raw 调试数据。
- 经济、BD、剧情三类数据都有稳定 id，便于收藏和最近浏览。
